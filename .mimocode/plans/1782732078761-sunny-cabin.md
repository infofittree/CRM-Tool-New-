# Plan: Add Product Information to Lead Data Entry & CRM Integration

## Current State

There's a `product_interest: String(255)` field on the Lead model, but it's broken:
- React DataEntry form has NO product field — leads created via web UI always have `product_interest = NULL`
- `LeadUpdate` schema omits `product_interest` — edits silently dropped
- `CRMService.EDITABLE_FIELDS` omits `product_interest` — backend ignores updates
- No product catalog, no dropdown, no validation

## Design Decision

**Two-table approach**: Create a `products` table + a `lead_products` junction table. This supports multi-product per lead in the future while keeping the current implementation simple (single product required).

The existing `product_interest` text field stays on the Lead model for backward compatibility (CSV imports, legacy data) but the new structured system takes priority.

## Files to Create

### 1. `database/models.py` — Add Product + LeadProduct models

```python
class Product(Base):
    __tablename__ = "products"
    id: int (PK, autoincrement)
    name: str (String(100), UNIQUE, NOT NULL)
    category: str (String(50), NOT NULL)  # "Spices", "Pulses", "Grains", "Herbs", "Seeds"
    is_active: bool (default=True)

class LeadProduct(Base):
    __tablename__ = "lead_products"
    lead_id: str (FK → leads.lead_id)
    product_id: int (FK → products.id)
    __table_args__ = (PrimaryKeyConstraint("lead_id", "product_id"),)
```

## Backend Changes Detail

### New endpoint: `GET /leads/filter-options` — Extend with products

Add a `product=true` query param that returns unique product names from the `products` table joined to leads.

### Lead creation flow with products:

```
LeadCreate.product_ids: [1, 3]  (e.g., Turmeric, Cumin)
    ↓
save_lead_from_entry()
    ↓
Lead record created
    ↓
INSERT INTO lead_products (lead_id, product_id) VALUES (lead_id, 1), (lead_id, 3)
    ↓
product_interest text field auto-populated from product names (backward compat)
```

### Lead update flow with products:

```
LeadUpdate.product_ids: [1, 5]  (e.g., change from Turmeric+Cumin to Turmeric+Black Pepper)
    ↓
edit_lead_fields()
    ↓
DELETE FROM lead_products WHERE lead_id = X
INSERT INTO lead_products (lead_id, product_id) VALUES (X, 1), (X, 5)
    ↓
product_interest text field updated
```
