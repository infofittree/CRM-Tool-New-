# Bug Fix Plan — 3 Issues

## Bug 1: Lead table filters only work on current page

**Root Cause:** `useLeads` fetches paginated data (25 per page). The filter dropdowns extract unique values from `data.items` (current page only), and `filteredItems` filters only that page's data. Country, Priority, Assigned, and Score filters are client-side on 25 rows.

**Fix:** Add backend query params for `country`, `priority`, and `score` so filtering happens server-side on the full dataset. The filter dropdowns will still use current page values (acceptable for UX), but the actual filtering will query the full database.

**Files to modify:**
- `sales_lead_system/api/routers/leads.py` — add `country`, `priority_level`, `lead_score_band` query params to `list_leads`
- `web/src/hooks/useLeads.ts` — pass new filter params to the API
- `web/src/pages/LeadManagement.tsx` — remove client-side `filteredItems`, pass filters to `useLeads`

---

## Bug 2: Default follow-up date should be 2 days after completion

**Root Cause:** In `ActivityWizard.tsx`, `followupDate` is initialized as `""` (empty string). The user must manually pick a date.

**Fix:** Initialize `followupDate` with a date 2 days from today. Calculate in the component body.

**Files to modify:**
- `web/src/components/ActivityWizard.tsx` — change `useState("")` to compute 2 days from now

---

## Bug 3: Leads should have an edit/modify option

**Root Cause:** LeadDetail page shows lead information but has no edit form. The backend already has `PUT /leads/{lead_id}` and `edit_lead_fields` service method.

**Fix:** Add an "Edit" button to LeadDetail that opens an edit form/modal. The form pre-fills with current lead data and calls the PUT endpoint on save.

**Files to modify:**
- `web/src/pages/LeadDetail.tsx` — add edit button, edit form state, save handler
- Use existing `api.put(/leads/${id}, ...)` endpoint
