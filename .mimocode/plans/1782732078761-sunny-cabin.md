# Plan: Open Lead Inside Task Modal (In-Modal Lead Brief)

## Current Flow
1. User clicks task → TaskWorkflowModal opens (line 236-240)
2. User clicks "Open Lead" → `navigate(/leads/${lead_id})` + `onClose()` → leaves Tasks page entirely
3. User must manually navigate back to Tasks page to resume

## Desired Flow
1. User clicks task → TaskWorkflowModal opens
2. User clicks "Open Lead" → Modal shows lead brief **inside the same modal** (no navigation)
3. User clicks "Back" → Returns to task details view
4. User can still click "Complete Task" from the task view

## Key Insight: No Extra API Call Needed

The `Task` type (api.ts:119-151) already carries most lead fields:
- `lead_id`, `company_name`, `contact_person`, `phone`, `email`, `country`
- `product_interest`, `interest_level`, `potential_deal_value`, `customer_requirements`
- `status`, `standard_status`, `score`, `band`, `lead_category`
- `assigned_to`, `lead_source` (via `reason`), `buyer_engagement_frequency`

This means the lead brief can render **immediately** from the task prop — zero loading states, zero fetch delay. Only the "Open Full Lead" escape hatch navigates to the full LeadDetail page.
