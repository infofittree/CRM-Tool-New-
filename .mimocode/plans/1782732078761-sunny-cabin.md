# Data Entry Follow-Up + ActivityWizard Redesign

## Part 1: Data Entry — Add Follow-Up Fields

### Current State
- `LeadCreate` schema already has `next_follow_up` (date), `last_discussion`, `next_action` fields
- Backend `save_lead_from_entry` already creates a follow-up if those fields are present
- **Gap**: Frontend `DataEntry.tsx` doesn't expose these fields — no way to schedule a follow-up during lead creation
- **Gap**: No "follow-up mode" (call, email, whatsapp, etc.) field exists

### Changes

**File: `sales_lead_system/api/schemas.py`**
- Add `followup_mode: str | None = None` to `LeadCreate` (values: "call", "email", "whatsapp", "sms", "meeting")

**File: `sales_lead_system/modules/crm_service.py`**
- In `save_lead_from_entry`, pass `followup_mode` to the follow-up payload as `mode`

**File: `web/src/pages/DataEntry.tsx`**
- Add two new fields after the Status/Priority row:
  - **Follow-Up Date** (date input)
  - **Follow-Up Mode** (select: Call, Email, WhatsApp, SMS, Meeting)
- Add these to the `defaultForm` state and `form` state
- Send `next_follow_up` and `followup_mode` in the POST body
- Group these in a "Schedule First Follow-Up" section with a divider

## Part 2: ActivityWizard — Redesign Step 3 (Next Action)

### Current State
- Step 3 shows a blue info box: "System will auto-create: Follow-Up Call" (derived from STATUS_SUGGESTED_ACTIONS)
- Has a date picker for follow-up date
- Has a notes textarea
- **Gap**: No way to choose the next follow-up mode (call, email, etc.)
- **Gap**: No structured way to record what was done in the current task session

### Changes

**File: `web/src/components/ActivityWizard.tsx`**

In Step 3, replace the current layout with:

1. **"What happened in this session?"** — A required textarea asking the salesperson to describe what was done. This replaces the generic "Notes" field and becomes the primary input. (Max 500 chars)

2. **"Schedule Next Follow-Up"** section:
   - **Follow-Up Mode** — Radio/select buttons: Call, Email, WhatsApp, SMS, Meeting (reuse ACTION_OPTIONS)
   - **Follow-Up Date** — Date picker (already exists)
   - Both required to proceed

3. **Auto-create preview** — Keep the blue info box but make it dynamic based on selected mode:
   - "Will create: Follow-Up Call on [date]"
   - "Will create: Check Response via Email on [date]"
   - etc.

4. Remove the old generic "Notes" textarea — the session notes replace it

### Backend Impact
- The `ActivityWizardRequest` schema already accepts `notes` and `followup_date`
- Add `next_followup_mode: str | None = None` to `ActivityWizardRequest`
- In `activity_engine.py`, pass the mode to the auto-created follow-up's `mode` field
