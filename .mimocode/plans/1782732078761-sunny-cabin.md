# Plan: Convert Inquiry Side Panel to Centered Modal

## Current State
The Inquiry module (`web/src/pages/InquiryPortal.tsx`, 708 lines) uses a right-side drawer (fixed, right-aligned, full-height) for inquiry details. The drawer has two role-specific content paths (Sales + Procurement) with the procurement commitment system inline.

## Target State
A centered modal (like `TaskWorkflowModal`) with portal rendering, scroll lock, and multi-view state machine. Procurement actions (commit/respond) render inline within the same modal — no stacking dialogs.

## Architecture

### New file: `web/src/components/InquiryWorkflowModal.tsx`

Follows the same pattern as `TaskWorkflowModal`:
- `createPortal(content, document.body)` — out of page DOM tree
- Lock `<main>` scroll (save/restore scrollTop)
- ESC key support (except during action views)
- 200ms fade transition between views
- Props: `{ inquiry: Inquiry; onClose: () => void; }`

**State machine:**
```
ModalView = "details" | "action" | "success"
```

- **details** — Shows inquiry info, timeline, response (if any), action buttons
- **action** — Procurement commitment form OR response form (replaces content inline)
- **success** — Confirmation message, auto-close after 1.5s

### Modified file: `web/src/pages/InquiryPortal.tsx`

- Remove the drawer JSX (lines 429-705)
- Remove drawer-related state: `committing`, `commitType`, `commitDate`, `commitResponse`, `respondText`
- Keep: `selectedInquiry` state + render `<InquiryWorkflowModal>` conditionally
- Keep: Create modal, list view, filters, pagination — all unchanged

### No backend changes

All API endpoints remain the same. The modal calls the same functions from `@/lib/inquiries.ts`.

## InquiryWorkflowModal — View: Action

When Procurement clicks "Respond" or Sales clicks "Reply":

The details content is replaced with the appropriate form (same modal, no new popup).

**Procurement paths:**
1. Commitment cards (Answer Now / BY_EOD / Will Take Time) → sets commitment type
2. Commitment form (textarea for ANSWER_NOW, date picker for WILL_TAKE_TIME)
3. Submit → calls `commitInquiry()` or `updateInquiry()` → transitions to success

**Sales path:**
1. Reply textarea → Submit → calls `updateInquiry({ response })` → success
