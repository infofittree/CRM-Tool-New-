# Critical Fix Plan — Three Non-Functioning Buttons

## Problem Statement
Three buttons are not functioning for the salesperson:
1. **Complete Task** button on Tasks tab
2. **Data Entry** button (New Lead navigation)
3. **Procurement inquiry** generation button

## Root Cause Analysis

After thorough investigation, the source code for all three buttons is correct:
- `Tasks.tsx` has `handleCompleteTask` with `wizardFollowupId` state
- `DataEntry.tsx` has working form submission
- `InquiryPortal.tsx` has working inquiry creation

The most likely cause is a **stale frontend deployment**. The deployed JS bundle on Railway may not contain the latest code changes. The server serves the old bundle which has outdated logic.

## Fix Strategy

**Step 1: Force a clean rebuild and redeploy**
- Delete old dist files
- Rebuild from scratch with `VITE_API_URL=/api/v1`
- Deploy to Railway
- Verify the bundle hash matches

**Step 2: Verify API connectivity from browser**
- Open https://crm-fittree.up.railway.app
- Open browser DevTools (F12)
- Check Console tab for errors
- Check Network tab for failed requests

**Step 3: Test each button**
- Complete Task: Open a task → click Complete Task → wizard should open
- Data Entry: Click New Lead → form should appear
- Procurement: Create inquiry → form should submit

## Files to rebuild
- `web/dist/` — clean and rebuild
- `web/src/pages/Tasks.tsx` — verify Complete Task logic
- `web/src/pages/DataEntry.tsx` — verify form submission
- `web/src/pages/InquiryPortal.tsx` — verify inquiry creation
