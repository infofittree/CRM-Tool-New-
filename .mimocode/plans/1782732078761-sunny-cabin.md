# Bug Fix Plan — Two Tasks Tab Issues

## Bug 1: Complete Task button not functioning

**Root Cause:** In `Tasks.tsx` line 471, the ActivityWizard is only rendered when `selectedTask.followup_id` exists:
```jsx
{showActivityWizard && selectedTask && selectedTask.followup_id && (
  <ActivityWizard followupId={selectedTask.followup_id} ... />
)}
```
If a lead has no follow-up record (e.g., newly created leads, or leads where the follow-up was deleted), `followup_id` is `null`. The button click sets `showActivityWizard(true)` but the wizard never renders because of the `selectedTask.followup_id` check.

**Fix:** When `followup_id` is null, create a temporary follow-up record on the backend before opening the wizard. Or, modify the wizard to accept a `leadId` instead of `followupId` and create the follow-up internally.

**Simpler fix:** Modify the wizard to work with `leadId` when `followupId` is null. The backend `complete-activity` endpoint already receives the `followupId` in the URL, so we need to either:
1. Create a quick follow-up record first, then open the wizard
2. Or change the wizard to accept `leadId` and create the follow-up as part of submission

**Recommended approach:** Create a quick follow-up record when the user clicks "Complete Task" and the lead has no followup_id. This keeps the existing wizard logic intact.

**Files to modify:**
- `web/src/pages/Tasks.tsx` — when clicking Complete Task and followup_id is null, POST to create a quick follow-up first, then open wizard

## Bug 2: Default 2-day follow-up date not visible

**Root Cause:** The 2-day default IS set in `ActivityWizard.tsx` (lines 115-119), but users never see it because:
1. The wizard doesn't open when followup_id is null (Bug 1 above)
2. The date picker is only shown in Step 3 of the wizard, which requires completing Steps 1 and 2 first

**Fix:** Once Bug 1 is fixed (wizard opens), the 2-day default will be visible in Step 3. Additionally, ensure the date picker is prominent and pre-filled correctly.

**Files to modify:**
- `web/src/components/ActivityWizard.tsx` — ensure date is visible and pre-filled (already done, just needs Bug 1 fix)
