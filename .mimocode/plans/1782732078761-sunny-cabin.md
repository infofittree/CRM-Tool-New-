# Bug Fix Plan — Salesperson Selector Duplicates

## Issue
The "View Analytics For" dropdown shows each person twice — once in uppercase (e.g., "KUSUM") and once in mixed case (e.g., "Kusum"). This is because the database has duplicate `full_name` entries from the CSV import with different casing.

## Root Cause
`crm_service.py:get_salespersons()` combines names from:
1. `users.full_name` table (has entries like "Kusum" and "KUSUM" from CSV import)
2. `leads.assigned_to` column (may also have different cases)

The method uses a `set()` for deduplication, but "Kusum" and "KUSUM" are different strings so both appear.

## Fix
**File:** `sales_lead_system/modules/crm_service.py` — line 85-92

Normalize all names to their original casing but deduplicate by lowercase comparison. Keep the first occurrence of each name (the properly cased one).

**Specific change:**
```python
def get_salespersons(self) -> list[str]:
    user_names = self.session.scalars(
        select(User.full_name).where(User.role.in_(["Salesperson", "Manager", "Admin"]), User.is_active.is_(True))
    ).all()
    assigned = self.session.scalars(select(Lead.assigned_to).where(Lead.assigned_to.is_not(None)).distinct()).all()
    
    # Deduplicate by lowercase comparison, keeping the first (properly cased) version
    seen = set()
    unique = []
    for name in [*user_names, *assigned]:
        if name and name.lower() not in seen:
            seen.add(name.lower())
            unique.append(name)
    
    return sorted(unique) or ["Unassigned"]
```
