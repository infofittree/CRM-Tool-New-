# Plan: Fix Duplicate Tasks in Active Queue

## Root Cause

The same lead appears in multiple task buckets (e.g., both "overdue" and "today") with the same followup_id. This happens because:
1. The task engine iterates over leads and generates tasks based on follow-up dates
2. For some leads, the `days_to` calculation can yield borderline values that place the task in different buckets on different iterations
3. The task engine processes up to 1000 leads and appends to bucket lists without deduplication

The fix: deduplicate the output by `lead_id` before returning, keeping only the first occurrence (highest priority score).

## Files to Modify

- `sales_lead_system/modules/task_engine.py` — Add deduplication after the main loop

## Implementation

After the main loop processes all leads and before sorting, deduplicate each bucket by `lead_id`:

```python
# Deduplicate: keep highest-priority task per lead in each bucket
def _dedup_by_lead(bucket):
    seen = set()
    result = []
    for t in bucket:
        if t["lead_id"] not in seen:
            seen.add(t["lead_id"])
            result.append(t)
    return result

overdue = _dedup_by_lead(overdue)
today_tasks = _dedup_by_lead(today_tasks)
upcoming = _dedup_by_lead(upcoming)
```

## Verification

1. Vaidehi's overdue count should drop (no more cross-bucket duplicates)
2. All API endpoints still work
3. Task completion still works
