# Bug Fix Plan — Tasks Tab: Drawer Position + Complete Button

## Issue 1: Drawer opens at top, user must scroll to see it

**Root Cause:** The drawer is `fixed top-0 right-0` which always opens at the viewport top. When the user clicks a task at the bottom of the list, the drawer appears at the top and the user must scroll up to see the header and action buttons.

**Fix:** Add a `useEffect` that scrolls the drawer content to the top when `selectedTask` changes. This ensures the header and action buttons are always visible when the drawer opens.

**File:** `web/src/pages/Tasks.tsx`
- Add a `ref` for the drawer content area
- Add `useEffect` to scroll to top when `selectedTask` changes

---

## Issue 2: Complete Task button still not functioning

**Root Cause:** In `handleCompleteTask`, when `followup_id` is null, we call:
```js
setSelectedTask({ ...selectedTask, followup_id: res.data.followup_id });
setShowActivityWizard(true);
```
React batches these state updates. The wizard renders with the OLD `selectedTask` (still `followup_id: null`), so the wizard condition `selectedTask.followup_id` fails and the wizard never opens.

**Fix:** Use a separate state variable `wizardFollowupId` to track the followup_id for the wizard, independent of `selectedTask`. The wizard will use this state variable instead of `selectedTask.followup_id`.

**File:** `web/src/pages/Tasks.tsx`
- Add `wizardFollowupId` state variable
- In `handleCompleteTask`: set `wizardFollowupId` and `showActivityWizard` together
- In wizard rendering: use `wizardFollowupId` instead of `selectedTask.followup_id`
- Clear `wizardFollowupId` when closing the wizard
